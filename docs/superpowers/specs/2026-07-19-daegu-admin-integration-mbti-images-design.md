# 대구 관리자 통합 및 MBTI 이미지 관리 설계

## 목표

`school`, `music`, `mbti`를 하나의 대구 운영 섹션으로 묶는다. 관리자는 `/daegu/admin`에서 한 번 로그인하고 세 카테고리 중 하나를 선택해 목록으로 이동하며, 각 목록의 공통 헤더에서 재로그인 없이 카테고리를 전환한다.

기존 `GET /mbti/[type]` JSON echo 라우트는 제거한다. `mbti`를 `library`, `nakdong`과 같은 정식 이미지 카테고리로 확장해 업로드 API, 관리자 목록, 썸네일, 공개 이미지 보기, 원본 다운로드, 삭제, 이용 기록과 만료 정리를 지원한다.

`library`와 `nakdong`의 로그인, 세션, 화면 흐름은 변경하지 않는다.

## 확정된 사용자 흐름

### 통합 로그인과 선택 화면

- 대구 관리자 진입점은 `/daegu/admin` 하나다.
- 비로그인 상태에서는 기존 로그인 폼 패턴을 사용한다. 아이디 저장과 로그인 유지 기능도 유지하되 저장 키는 대구 섹션 단위로 사용한다.
- 로그인 요청은 대구 전용 인증 API로 전송하고 `DAEGU_ADMIN_ID`, `DAEGU_ADMIN_PASSWORD`를 검증한다.
- 로그인 성공 후 같은 `/daegu/admin` 주소를 다시 표시하면 다음 세 선택 카드가 보인다.
  - 나만의 학교 만들기
  - 나만의 음악 만들기
  - MBTI
- 선택 카드는 각각 `/school/admin`, `/music/admin`, `/mbti/admin`으로 이동한다.

### 목록 전환

- 세 카테고리의 목록 주소는 기존 동적 관리자 주소 형식인 `/{category}/admin`을 유지한다.
- 목록 상단에 대구 공통 헤더 바를 표시한다.
- 헤더 바에는 `나만의 학교 만들기`, `나만의 음악 만들기`, `MBTI` 링크가 있고 현재 카테고리를 시각적·접근성 속성으로 구분한다.
- 좁은 화면에서는 링크와 기존 목록 동작이 겹치지 않도록 자연스럽게 줄바꿈한다.
- 기존 이용 기록, 페이지 크기, 삭제, 로그아웃 기능은 유지한다.
- 대구 목록에서 로그아웃하면 공용 대구 세션을 만료시키고 `/daegu/admin`으로 이동한다.

기존 작업 트리에서 사용자가 변경한 `나만의 음악 만들기`, `나만의 학교 만들기` 라벨은 보존한다. 기존 테스트의 짧은 라벨 기대값만 새 라벨과 일치하도록 갱신한다.

## 라우팅 구조

### 신규·변경 라우트

- `GET /daegu/admin`: 비로그인 시 통합 로그인 폼, 로그인 시 카테고리 선택 화면
- `POST /api/daegu/auth/login`: 대구 공용 자격증명 검증 및 공용 세션 발급
- `POST /api/daegu/auth/logout`: 공용 세션 만료
- `GET /mbti/admin`: MBTI 이미지 관리자 목록
- `GET /mbti/admin/usage`: MBTI 이용 기록
- `GET /mbti/[uid]`: MBTI 공개 이미지 확인
- `/api/mbti/images*`: 기존 공통 이미지 API를 통한 업로드, 목록, 파일, 썸네일, 다운로드와 삭제

### 리디렉션과 라우트 우선순위

- 비로그인 상태로 `/school/admin`, `/music/admin`, `/mbti/admin` 또는 각 이용 기록 페이지에 접근하면 `/daegu/admin`으로 리디렉션한다.
- 로그인 상태에서는 위 주소에 직접 접근할 수 있다. 따라서 북마크와 헤더 링크를 유지할 수 있다.
- 정적 `/daegu/admin` 라우트는 동적 `/[category]/admin`보다 우선한다. `daegu`는 이미지 카테고리 목록에 추가하지 않는다.
- `src/app/mbti/[type]/route.ts`는 제거한다. 그러면 `/mbti/[uid]`는 공통 `src/app/[category]/[uid]/page.tsx`가 처리한다.
- `/mbti/[uid]`는 더 이상 `{ "type": "..." }` JSON을 반환하지 않는다.

## 인증과 권한 모델

현재 단일 세션 쿠키 이름은 유지한다. 세션 서명 payload가 소유하는 범위를 이미지 카테고리와 관리자 그룹으로 구분한다.

- `library` 세션은 `library`만 허용한다.
- `nakdong` 세션은 `nakdong`만 허용한다.
- `daegu` 세션은 `school`, `music`, `mbti` 관리자 페이지와 보호 API를 허용한다.
- 대구 세션은 `library`, `nakdong`을 허용하지 않는다.
- `library`나 `nakdong` 세션은 대구 섹션을 허용하지 않는다.

인증 모듈에는 카테고리별로 필요한 관리자 범위를 반환하는 작은 정책 함수를 둔다. 페이지와 API는 같은 정책을 사용해 서로 다른 권한 판정을 만들지 않는다. 기존 `library`와 `nakdong` 로그인 폼 및 인증 API는 현재 동작을 유지한다.

대구 목록의 이미지 목록·삭제·이용 기록 API도 공용 대구 세션을 검증한다. 공개 이미지 파일, 썸네일과 다운로드 API의 기존 공개 범위는 변경하지 않는다. 업로드 API는 기존과 같이 `UPLOAD_API_TOKEN`을 사용한다.

## 컴포넌트 경계

### 대구 관리자 페이지

서버 페이지가 공용 대구 세션을 확인한다. 비로그인 상태에서는 대구 전용 범위를 전달받은 로그인 폼을 렌더링하고, 로그인 상태에서는 단순한 카테고리 선택 컴포넌트를 렌더링한다. 선택 컴포넌트는 데이터 요청을 하지 않고 세 목록 링크만 소유한다.

### 공통 대구 헤더

대구 헤더는 현재 카테고리만 입력받아 세 링크와 활성 상태를 렌더링한다. 이미지 목록의 페이징, 삭제, 음악 재생 상태를 소유하지 않는다. 이 경계를 통해 헤더 전환 동작을 독립적으로 테스트한다.

### 이미지 목록과 뷰어

`ImageList`는 기존 동작을 유지하며 대구 카테고리일 때 공통 헤더와 대구 로그아웃 경로를 사용한다. `music`만 오디오 UI를 사용한다. `mbti`는 `library`, `school`과 같은 이미지 썸네일과 표준 공개 뷰어를 사용한다.

## MBTI 데이터와 저장 흐름

`mbti`를 정식 `Category`에 추가한다. 이로써 기존 이미지 서비스, R2 키 생성, D1 저장소, 이용 기록, 만료 정리가 별도 복제 없이 MBTI에도 적용된다.

- 업로드: `POST /api/mbti/images`와 기존 업로드 토큰 사용
- 저장 키: `images/mbti/{uid}/{filename}`
- 썸네일 키: `images/mbti/{uid}/thumbnail.webp`
- 공개 보기: `/mbti/{uid}`
- 관리자 목록: `/mbti/admin`

MBTI는 이미지 카테고리이므로 기존 이미지 MIME 검증과 썸네일 생성을 사용한다. 음악 카테고리의 오디오 예외는 MBTI에 적용하지 않는다.

## D1 마이그레이션

이미 배포된 `0003_expand_categories.sql`은 수정하지 않는다. 새 `0004_add_mbti_category.sql`을 추가한다.

마이그레이션은 `images`와 `usage_records`를 `library`, `nakdong`, `music`, `school`, `mbti`를 허용하는 CHECK 제약으로 재생성한다. 기존 행의 ID와 모든 열을 복사하고, 기존 고유 인덱스와 조회 인덱스를 동일한 이름과 정렬로 복원한다.

런타임의 누락 테이블 부트스트랩 스키마도 같은 다섯 카테고리를 허용한다. 마이그레이션 테스트는 두 테이블의 CHECK 제약, 행 복사와 인덱스 복원을 검사한다.

## 오류 처리

- 잘못된 대구 로그인은 기존 로그인 오류 형식을 표시한다.
- 로그인·로그아웃 네트워크 오류는 현재 관리자 UI 패턴의 메시지를 표시하고 사용자가 다시 시도할 수 있게 한다.
- 인증되지 않은 보호 API는 기존처럼 HTTP 401을 반환한다.
- 인증되지 않은 대구 관리자 페이지는 `/daegu/admin`으로 리디렉션한다.
- 알 수 없는 카테고리는 기존 404 또는 카테고리 파싱 오류 동작을 유지한다.
- 없는 MBTI UID는 기존 공개 이미지 404 화면을 사용한다.
- 마이그레이션은 기존 데이터와 인덱스를 보존하지 못하는 형태로 적용하지 않는다.

## 테스트 전략과 완료 기준

구현은 테스트 우선으로 진행한다.

1. 카테고리 테스트에서 `mbti` 파싱, 라벨과 사용자가 변경한 긴 대구 라벨을 먼저 고정한다.
2. 인증 테스트에서 `daegu` 세션이 `school`, `music`, `mbti`에 유효하고 `library`, `nakdong`에는 유효하지 않음을 검증한다.
3. 대구 로그인 API가 공용 자격증명을 검증하고 공용 세션을 발급하는 테스트를 추가한다.
4. `/daegu/admin`이 인증 상태에 따라 로그인 폼 또는 세 카테고리 선택 화면을 렌더링하는 테스트를 추가한다.
5. 대구 카테고리 관리자 페이지의 비로그인 리디렉션과 로그인 상태 접근을 검증한다.
6. 공통 헤더의 세 링크, 현재 카테고리 표시, 목록 전환과 대구 로그아웃 동작을 검증한다.
7. MBTI 업로드, 목록, 썸네일, 공개 뷰어, 다운로드와 삭제의 대표 API·컴포넌트 테스트를 추가한다.
8. 기존 MBTI echo E2E 테스트를 이미지 경로 동작과 JSON echo 제거를 검증하는 테스트로 교체한다.
9. 새 `0004` 마이그레이션이 다섯 카테고리를 허용하고 기존 행·인덱스를 보존하는지 검증한다.
10. `library`, `nakdong`, 음악 오디오 재생, 기존 로그인 흐름의 회귀 테스트를 유지한다.

완료 전에 전체 단위 테스트, TypeScript 타입 검사, ESLint, 프로덕션 빌드와 관련 Playwright E2E를 실행한다. 모든 명령이 성공하고 작업 트리에 의도하지 않은 파일 변경이 없어야 완료로 판단한다.

## 범위 제외

- `library`, `nakdong`을 `/daegu/admin`에 통합하지 않는다.
- 대구 카테고리의 공개 이미지 경로를 `/daegu/...` 아래로 이동하지 않는다.
- MBTI 유형 유효성 검사나 16개 유형별 별도 비즈니스 규칙을 추가하지 않는다.
- 이미지 업로드를 위한 새로운 관리자 폼을 추가하지 않는다. 기존 API 업로드 계약을 그대로 사용한다.
- 사용자 계정 DB, 역할 관리 화면 또는 다중 사용자 권한 시스템을 도입하지 않는다.
