# Music 관리자 오디오 문구 설계

## 목표

`music` 관리자 목록에서 이미지 자산을 뜻하는 화면 문구를 오디오 자산에 맞게 변경한다. `library`, `nakdong`, `school` 관리자 목록의 기존 이미지 문구는 유지한다.

## 변경 범위

공용 `ImageList` 컴포넌트 안에서 현재 카테고리가 `music`인지 확인해 다음 문구를 분기한다.

- `업로드 이미지` → `업로드 오디오`
- `총 N개 이미지` → `총 N개 오디오`
- `등록된 이미지가 없습니다.` → `등록된 오디오가 없습니다.`
- `이미지를 삭제할까요?` → `오디오를 삭제할까요?`

삭제 동작, 빈 목록 설명, 재생 UI, API 경로와 다른 카테고리의 화면은 변경하지 않는다.

## 구현 방식

`ImageList`에서 `category === "music"`을 기준으로 자산 명칭을 계산하고 네 문구에 재사용한다. 카테고리 설정 구조 확장이나 music 전용 컴포넌트 분리는 이번 범위에 필요하지 않다.

## 테스트

- music에 항목이 있을 때 제목과 총 개수가 오디오 문구로 표시되는지 확인한다.
- music 빈 목록이 오디오 문구를 표시하는지 확인한다.
- music 삭제 확인창이 오디오 문구를 표시하는지 확인한다.
- library의 제목, 총 개수, 빈 목록과 삭제 확인창이 기존 이미지 문구를 유지하는지 확인한다.
